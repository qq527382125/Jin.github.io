Electron搭建node桌面应用

Electron, node 2018-04-30

Electron搭建node桌面应用

2018-04-30

Electron, node

使用node开发桌面应用吧

介绍

Electron可以用javascript调用操作系统的API来创建桌面应用（如MACOS和WINOWS应用）。
直接采用web页面作为它的GUI,可以看成一个精简版的Chromium浏览器。

本质上，Electron应用是一个Node.js应用程序， 应用启动的入口是一个与node相同的package.json 文件。

基本的Electron应用基本的目录结构如下：

app/
  ├── package.json
  ├── main.js
  └── index.html

通常想要创建一个Electron应用，找个空的文件夹，运行npm init ,会生成package.json文件，其中main字段表示的脚本为应用的启动脚本，它会在主进程中执行。

{
  "name":"app",
  "version": "0.0.1",
  "main" : "main.js"
}

如果package.json没有main字段，那么就会加载index.js文件(node的规则)。
node应用需要一个start脚本来指引node去执行当前的package.

{
    "name": "your-app",
    "version": "0.1.0",
    "main": "main.js",
    "scripts": {
      "start": "node ."
    }
  }

而要将普通的node应用转换为Electron应用很简单，只要将start从node时改成electron便可。

{
    "name": "your-app",
    "version": "0.1.0",
    "main": "main.js",
    "scripts": {
      "start": "electron ."
    }
  }

要创建一个Electron应用首先要安装Electron模块。
npm install --save-dev electrpm

--save-dev 把依赖包名称添加到 package.json文件的dependencies键下。

一个简易的Electron

electron模块包含了Electron提供的所有API和功能。electron模块所提供的功能都是通过命名空间暴露出来的。
比如electron.app负责管理应用的生命周期，electron.BrowserWindow类负责创建窗口.想使用这些功能，就需要将模块引入。

如const electron = require('electron')

比如一个简单的main.js文件，它将在应用程序准备就绪后打开一个窗口。

const {app,BrowserWindow} = require('electron')
    const path = require('path')
    const url = require('url')

    function createWindow() {
        //创建一个窗口
        win = new BrowserWindow({width:800,height: 600})
        //加载应用的index.html
        win.loadURL(url.format({
            pathname:path.join(__dirname,'index.html'),
            protocol: 'file',
            slashes:true
        }))
    }

    app.on('ready',createWindow)

创建完窗口后，处理程序中可能会遇到所有系统事件，我们完善下代码，添加功能：打开开发者工具，处理窗口关闭事件，在macosyoghurt点击dock上图标时重建窗口。

const {app, BrowserWindow} = require('electron')
  const path = require('path')
  const url = require('url')

  // 保持一个对于 window 对象的全局引用，如果你不这样做，
  // 当 JavaScript 对象被垃圾回收， window 会被自动地关闭
  let win

  function createWindow () {
    // 创建浏览器窗口。
    win = new BrowserWindow({width: 800, height: 600})

    // 然后加载应用的 index.html。
    win.loadURL(url.format({
      pathname: path.join(__dirname, 'index.html'),
      protocol: 'file:',
      slashes: true
    }))

    // 打开开发者工具。
    win.webContents.openDevTools()

    // 当 window 被关闭，这个事件会被触发。
    win.on('closed', () => {
      // 取消引用 window 对象，如果你的应用支持多窗口的话，
      // 通常会把多个 window 对象存放在一个数组里面，
      // 与此同时，你应该删除相应的元素。
      win = null
    })
  }

  // Electron 会在初始化后并准备
  // 创建浏览器窗口时，调用这个函数。
  // 部分 API 在 ready 事件触发后才能使用。
  app.on('ready', createWindow)

  // 当全部窗口关闭时退出。
  app.on('window-all-closed', () => {
    // 在 macOS 上，除非用户用 Cmd + Q 确定地退出，
    // 否则绝大部分应用及其菜单栏会保持激活。
    if (process.platform !== 'darwin') {
      app.quit()
    }
  })

  app.on('activate', () => {
    // 在macOS上，当单击dock图标并且没有其他窗口打开时，
    // 通常在应用程序中重新创建一个窗口。
    if (win === null) {
      createWindow()
    }
  })

  // 在这个文件中，你可以续写应用剩下主进程代码。
  // 也可以拆分成几个文件，然后用 require 导入。

最后创建你想展示的index.html

<!DOCTYPE html>
  <html>
    <head>
      <meta charset="UTF-8">
      <title>Hello World!</title>
    </head>
    <body>
      <h1>Hello World!</h1>
      We are using node <script>document.write(process.versions.node)</script>,
      Chrome <script>document.write(process.versions.chrome)</script>,
      and Electron <script>document.write(process.versions.electron)</script>.
    </body>
  </html>

启动应用

在创建并初始化完成main.js，index.html和package.json之后，就可以在当前工程的根目录执行npm start （运行启动的脚本）命令来启动刚刚编写好的Electron程序了。

Demo

可以上：https://github.com/electron/electron-quick-start 下载一个实例仓库。

运行 npm init ，npm根据package.json下载所需的依赖。(依赖可能比较大，可以使用淘宝NPM镜像源)

# 克隆这仓库
  $ git clone https://github.com/electron/electron-quick-start
  # 进入仓库
  $ cd electron-quick-start
  # 安装依赖库
  $ npm install
  # 运行应用
  $ npm start

安装了好多node模块。

运行时如果出现

fs.js:1669
binding.lstat(baseLong);
^

很可能是文件路径问题，移动到不包含ASCII的文件夹下

electron桌面应用在开发环境下能正常跑通了，现在如何将它变成windows运行的exe或者macos运行的app呢。

需要专用的打包工具electron-packager 这个工具在今后也会使用到，所以推荐安装在全局。

npm install electron-packager -g

把当前文件目录下的资源（.）命名为bobo打包到父级的electron文件夹。(父级没有electron文件夹的话，会自动创建)

此时electron-packager就会自动判别当前的操作系统打包对应的文件，例如windows系统下就会打包成.exe格式。

electron-packager . bobo --out ../electron

其实被打包的文件被放在了打包号后的文件夹下resources/app文件夹下，要注意的是，如果在打包的时候,用--save-dev 安装的模块不会被打包进去。如果需要那些模块，我们可以将需要用的第三方模块压缩成压缩包放在原来的目录下，这样就会被一起打包到app下，打包之后解压到node_moudle下就就可以了。

升级应用

平常的桌面应用要升级的时候，一般需要下载完整的安装包，但是electron桌面应用具体的业务代码实际上都是在app文件夹下，其他的东西是桌面应用的浏览器内核和主线程，如果我们只是调整UI和交互只涉及前端代码的部分，我们只需要更新app下的文件就可以了。比如原文件有个js代码要更新，只需要发布个安装包将这个js文件替换掉就好。

（如果原来的js文件是外链的话就更简单了，直接将外链的源修改掉就好了）

我们还可以添加自动检测是否有新版本更新，已经在线升级，来更新CSS,HTML,JS等静态资源。
